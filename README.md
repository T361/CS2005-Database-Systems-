## 📌 Overview
This repository contains my coursework, SQL scripts, Entity-Relationship (ER) models, and database architecture projects for **CS2005: Database Systems (3+1)** at the **National University of Computer and Emerging Sciences (FAST-NU)**. 

This course explores the rigorous theory and practical application of data storage, retrieval, and management. The `+1` represents the hands-on lab component, focusing on writing complex SQL queries, designing normalized database schemas, and building robust backend data architectures.

---

## 🛠️ Core Concepts Covered

* **Relational Data Modeling:** Designing Entity-Relationship Diagrams (ERDs) and mapping them to relational schemas.
* **Database Normalization:** Eliminating data redundancy and anomalies through 1NF, 2NF, 3NF, and BCNF.
* **Advanced SQL:** Complex joins, subqueries, grouping, aggregation, and window functions.
* **Database Programming:** Writing stored procedures, triggers, views, and utilizing PL/SQL or T-SQL.
* **Transaction Management & Indexing:** Ensuring ACID properties, managing concurrency control, and optimizing query performance via indexes.

---

## 📂 Repository Structure

The repository is organized into weekly labs and architectural design assignments:

* `Labs/`: Weekly SQL scripts focusing on DDL, DML, DCL, and complex data querying.
* `Assignments/`: Comprehensive database design tasks including ER modeling and normalization proofs.
* `Project/`: The final course project involving the full lifecycle of designing, building, and querying a massive relational database.

---

## 🚀 How to Run

The scripts in this repository are written in SQL and can be executed in Relational Database Management Systems (RDBMS) like Microsoft SQL Server, MySQL, or PostgreSQL.

1. Clone the repository: `git clone https://github.com/T361/CS2005-Database-Systems-.git`
2. Open your preferred SQL IDE (e.g., SSMS, DataGrip, pgAdmin).
3. Load the `.sql` schema files first to generate the tables and relationships, then run the corresponding query scripts.
